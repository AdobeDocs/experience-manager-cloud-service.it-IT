# Innovazioni principali di Adobe Experience Manager Forms {#latest-innovations}

Alcune delle principali innovazioni di Adobe Experience Manager Forms includono:

| Innovazione | Dettagli |
|---|---|
| Forms adattivo headless | Creare e gestire [Forms adattivo headless](https://experienceleague.adobe.com/docs/experience-manager-headless-adaptive-forms/using/overview.html?lang=it) nella piattaforma Adobe Experience Manager. Consente agli sviluppatori di creare, pubblicare e gestire moduli interattivi a cui è possibile accedere e con cui interagire tramite API, anziché tramite un’interfaccia utente grafica tradizionale. <br/> <br/> Questi moduli sono progettati per essere inviati senza la necessità di un’interfaccia HTML tradizionale. In altre parole, ti consentono di inviare i dati del modulo a livello di programmazione tramite un’API o un codice backend con e senza elementi del modulo visibili sul front-end. <br/> <!-- ![Headless adaptive form working](https://experienceleagueadobe.com/docs/experience-manager-headless-adaptive-forms/assets/how-headless-adaprive-forms-work.png?) --><br/> I moduli headless sono utili in vari scenari, ad esempio durante la creazione di applicazioni a pagina singola, app web progressive o applicazioni mobili, in cui un’interfaccia HTML tradizionale potrebbe non essere necessaria o pratica. Consentendo agli sviluppatori di inviare i dati del modulo direttamente tramite API o codice di back-end, i moduli headless consentono di semplificare i flussi di lavoro e migliorare le prestazioni complessive delle applicazioni web. |
| Componenti core | Il [Componenti core Forms adattivi](https://experienceleague.adobe.com/docs/experience-manager-core-components/using/adaptive-forms/introduction.html#features) sono un set di 24 componenti open-source conformi a BEM basati sui componenti core WCM di Adobe Experience Manager. Sono progettati appositamente per la creazione di moduli adattivi, ossia moduli che si adattano al dispositivo, al browser e alle dimensioni dello schermo dell’utente. <br/> <br/> Questi componenti possono essere utilizzati per creare esperienze eccezionali di acquisizione e registrazione dei dati fornendo un’ampia gamma di opzioni per i campi modulo, compresi campi di testo, caselle di controllo, menu a discesa e altro ancora. Includono anche funzioni quali convalida, logica condizionale e progettazione reattiva, che possono essere utilizzate per creare moduli intuitivi e facili da usare. <br/> ![funzionalità dei Componenti core](https://experienceleague.adobe.com/docs/experience-manager-cloud-service/assets/sample-core-components-based-adaptive-form.png?)<br/>  Inoltre, poiché questi componenti sono open-source, gli sviluppatori possono personalizzarli ed estenderli facilmente in base alle esigenze specifiche della loro organizzazione. Questi componenti sono inoltre basati sulla metodologia BEM, che garantisce la scalabilità e la manutenzione dei componenti. |
| Connettore Microsoft Power Automate | AEM Forms Power Automate Connector consente di integrare Adobe Experience Manager (AEM) Forms con Microsoft Power Automate (precedentemente noto come Microsoft Flow). Power Automate è un servizio basato su cloud che consente di creare flussi di lavoro automatizzati tra diverse applicazioni e servizi.  <br/> <br/> Con il connettore AEM Form Power Automate puoi creare flussi di lavoro che si attivano automaticamente in base all’invio di un modulo adattivo. Ad esempio, puoi creare un flusso di lavoro che invia automaticamente una notifica e-mail a una persona specifica quando un utente invia un modulo o crea un’attività in Microsoft Planner quando un utente completa un modulo.  <br/> ![icona del connettore microsoft PowerAutomate](https://powerusers.microsoft.com/t5/image/serverpage/image-id/182924i17C4BEA1C045D731/image-size/large/is-moderation-mode/true?v=1.0&amp;px=999) <br/> Il connettore AEM Forms Power Automate è un potente strumento che consente di automatizzare e integrare il Forms adattivo con altre applicazioni e servizi collegati a Microsoft Power Automate, consentendoti di lavorare con una gamma più ampia di strumenti. Puoi creare flussi di lavoro personalizzati in base alle tue esigenze, con la possibilità di aggiungere azioni, condizioni e trigger personalizzati. Inoltre, Power Automate fornisce analisi dettagliate e rapporti, consentendo di monitorare e ottimizzare i flussi di lavoro nel tempo. |
| Connettori di archiviazione Microsoft | Connettori di archiviazione AEM Forms Microsoft per <a href="https://experienceleague.adobe.com/docs/experience-manager-cloud-service/content/forms/adaptive-forms-authoring/authoring-adaptive-forms-foundation-components/configure-submit-actions-and-metadata-submission/configuring-submit-actions.html#submit-to-sharedrive">OneDrive</a>, <a href="https://experienceleague.adobe.com/docs/experience-manager-cloud-service/content/forms/adaptive-forms-authoring/authoring-adaptive-forms-foundation-components/configure-submit-actions-and-metadata-submission/configuring-submit-actions.html?#submit-to-sharedrive"> SharePoint </a> e <a href="https://experienceleague.adobe.com/docs/experience-manager-cloud-service/content/forms/adaptive-forms-authoring/authoring-adaptive-forms-foundation-components/configure-submit-actions-and-metadata-submission/configuring-submit-actions.html?#submit-to-azure-blob-storage"> Archiviazione BLOB di Azure </a> sono connettori che consentono di integrare Adobe Experience Manager (AEM) Forms con Microsoft OneDrive e SharePoint. Con questo connettore, puoi caricare file di dati e allegati in OneDrive e SharePoint direttamente da Adaptive Forms. <br/> ![icona di microsoft onedrive e sharepoint](/help/forms/assets/onedrive-and-sharepoint.jpg) <br/>OneDrive e SharePoint possono integrarsi con altre applicazioni aziendali, quali sistemi CRM, software di contabilità e strumenti di gestione dei progetti. Ciò consente di semplificare i processi aziendali, ridurre l&#39;immissione manuale dei dati e migliorare l&#39;efficienza complessiva. |
| Interfaccia utente procedura guidata | L’interfaccia utente della procedura guidata per Forms adattivo è uno strumento utile per creare moduli adattivi in modo rapido e semplice. L’interfaccia intuitiva e le opzioni di personalizzazione lo rendono accessibile a tutti gli utenti, indipendentemente dal loro livello di competenza tecnica. <br/> <br/> L’interfaccia utente della procedura guidata semplifica il processo di creazione dei moduli adattivi, guidando gli utenti nelle varie fasi del processo. L’interfaccia utente della procedura guidata è divisa in più schede, ciascuna delle quali fornisce chiaramente le opzioni per configurare un modulo adattivo. L’autore di un modulo avanza attraverso le schede in modo lineare per selezionare opzioni quali modelli, azioni di invio e origini dati per i componenti del modulo. <br/> ![immagine della procedura guidata per la creazione di moduli](/help/release-notes/assets/wizard.png) <br/>L’interfaccia della procedura guidata semplifica il processo di individuazione di tutte le opzioni essenziali per un modulo adattivo e semplifica la creazione dei moduli, anche per gli utenti che non hanno familiarità con la tecnologia. |
| Adobe Analytics con Experience Cloud Setup Automation per Forms | L’analisi dei moduli può fornire informazioni utili sulle prestazioni dei moduli misurando il coinvolgimento degli utenti, ottimizzando i tassi di conversione, monitorando le prestazioni dei moduli e migliorando l’esperienza utente.  Monitorando il comportamento degli utenti e i feedback, Analytics può identificare le aree del modulo che causano frustrazione o confusione, guidando i miglioramenti alla progettazione e alle funzionalità del modulo. <br/> <br/> Utilizzando puoi [abilitare Adobe Analytics con Experience Cloud Setup Automation](https://experienceleague.adobe.com/docs/experience-manager-cloud-service/content/forms/integrate/services/enable-adobe-analytics-adaptive-form-using-experience-cloud-setup-automation.html) con un paio di pulsanti. Consente di collegare AEM Forms as a Cloud Service con i tag Experienci Platform e Adobe Analytics per acquisire e tenere traccia delle metriche delle prestazioni per i moduli pubblicati. <br/> <br/> ![Report Adobe Analytics sul coinvolgimento degli utenti dei moduli adattivi](/help/forms/assets/forms-analytics-report.png) <br/><br/> Forms as a Cloud Service fornisce un rapporto OOTB di Adobe Analytics. Consente di comprendere facilmente le prestazioni dei moduli. Le metriche a livello di modulo forniscono informazioni approfondite sulle prestazioni del modulo per più indicatori di prestazioni chiave (KPI, Key Performance Indicator), come rappresentazioni, visitatori, invii e tempo medio di riempimento. <br/> <br/> Fornisce inoltre dettagli sul numero medio di accessi degli utenti alla guida contestuale per i campi nel pannello, per aiutarti a verificare i campi in base ai quali gli utenti si arrestano e cercano informazioni prima di fornire informazioni. Puoi semplificare ulteriormente questi campi o aiutare i contenuti a migliorare le conversioni. |
| Consegna Edge | Con Edge Delivery, i dipendenti di tutta l’azienda possono creare moduli con gli strumenti preferiti, ad esempio Microsoft Excel e Google Sheets, senza la necessità di una formazione approfondita sui nuovi flussi di lavoro dell’interfaccia utente. <br/> <br/> Inoltre, Edge Delivery è progettato per caricare moduli con una velocità senza pari, ottenendo un punteggio quasi perfetto di Google Lighthouse sulle pagine live, garantendo prestazioni e esperienza utente ottimali.<br/> <br/> ![strumenti di componibilità di nuova generazione per creare moduli](/help/forms/assets/web-vitals.jpeg) Inoltre, la distribuzione dei moduli non è mai stata più semplice o veloce, grazie agli strumenti semplici forniti da Edge Delivery. Con un tempo di sviluppo minimo richiesto, puoi semplificare facilmente il percorso per andare &quot;live&quot; e offrire esperienze di modulo eccezionali ai tuoi clienti. |
